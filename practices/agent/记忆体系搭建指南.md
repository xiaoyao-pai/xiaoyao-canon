# AI 记忆体系搭建指南

> 来源：逍遥派经验沉淀 | 脱敏版

## 核心问题

AI 是无状态的——每次对话从零开始，昨天踩过的坑今天又踩。

## 四层配置架构

| 层级 | 作用 | 类比 |
|------|------|------|
| **Rules（规则层）** | 告诉 AI 你是谁、工作方式、输出偏好 | 新员工入职培训 |
| **Skills（技能层）** | 特定场景的操作手册 | 标准 SOP |
| **Agents（代理层）** | 自动化执行者 | 专人专岗 |
| **Knowledge（模块层）** | 可复用的模板和数据 | 知识库 |

## Rules 文件设计要点

- 保持精简（≤150 行），详情拆到 knowledge/ 按需加载
- 用 `alwaysApply: true` 让规则每次对话自动生效
- 核心内容：职业背景、沟通偏好、工作风格、工具偏好
- 不要把所有东西都塞进 Rules——会导致 Token 消耗飙升且效果变差

## 记忆修剪原则（做梦机制）

- 每个记忆文件 ≤ 200 行
- 超出部分降级到子文件或归档
- 定期整理：去重、删矛盾、相对日期转绝对日期
- 冗长内容拆分，索引文件只保留指针

## 踩坑经验

- Rules 5 份砍到 2 份，Token 消耗降 52%，效果反而更好
- 配置不是越多越好，是"该出现时出现，不该出现时闭嘴"
- Skill 按需加载比全量注入高效得多
