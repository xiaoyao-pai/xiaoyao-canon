# 代码安全诊断实战

> 来源：逍遥派经验沉淀 | 脱敏版

## 为什么需要安全诊断

Vibe Coding 的速度优势是真实的，但安全债也是真实的。快速迭代中容易忽略安全问题。

## 诊断维度

让 AI 按以下 8 个维度做全量代码审计：
1. **安全**：CORS、密钥管理、输入校验
2. **认证授权**：JWT、端点认证、权限校验
3. **数据校验**：输入长度、类型、SQL 注入
4. **可靠性**：连接池、异步异常、重试机制
5. **性能**：N+1 查询、索引、缓存
6. **Schema 一致性**：数据库和代码的字段/约束是否对齐
7. **部署配置**：Dockerfile 用户权限、环境变量
8. **前端安全**：敏感信息存储、API Key 泄露

## 分批修复策略

| 批次 | 标准 | 时间 |
|------|------|------|
| **第一批** | 不修就裸奔（CORS 全开、JWT 缺失等） | 当天 |
| **第二批** | 有隐患但可控（弱密码、异常丢失等） | 一周内 |
| **第三批** | 锦上添花（性能优化、日志规范） | 迭代中 |

## 常见安全问题清单

- CORS `allow_origins=["*"]` → 改为白名单
- JWT secret 用默认值 → 非 debug 模式拒绝启动
- API 端点无认证 → 加 JWT + userId 校验
- Dockerfile 用 root → 创建 appuser
- 数据库连接池无锁 → asyncio.Lock + double-check
- Schema 和代码的 CHECK 约束不一致 → 同步更新
- 前端残留 API Key → 迁移到后端代理

## 关键原则

- 一个文件一个文件地修，每改完跑一次 linter
- 诊断报告存下来当"病历"，每次修复更新状态
- 写代码和审代码最好不在同一个对话里——新对话更客观
