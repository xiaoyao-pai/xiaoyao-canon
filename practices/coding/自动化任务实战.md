# 自动化任务实战经验

> 来源：逍遥派经验沉淀 | 脱敏版

## 什么时候该用自动化

当你发现一个流程满足以下条件：
- 步骤固定、每次差不多
- 需要定期重复执行
- 每个步骤都不难，但串起来耗时

## 自动化任务设计要点

- prompt 要自给自足——用户不在场，任务要自己能跑完
- 如果需要判断"做还是不做"，在 prompt 里写清楚判断条件
- 失败时记录错误但不要阻断——文件已保存本地不会丢
- 每次执行写日志到 memory 文件，方便下次知道上次做了什么

## 踩坑经验

- **子智能体不适合生成长文本（3000+ 字）**：会因 idle timeout 中断，长文本任务保留在主 Agent
- **自动化任务超时**：扫描 6+ 工作空间 + 写 3000-6000 字文章太重，考虑拆分步骤
- **重复创建**：用固定 ID 创建任务（INSERT OR IGNORE），防止重复安装时创建重复任务
- **工作空间路径**：自动化任务的 cwds 必须是实际存在的目录

## 经验提炼自动化

数据源优先级：
1. sessions 数据库（最完整，记录所有对话）
2. brain/overview.md（有产出的对话才有）
3. .codebuddy/memory/ 日期文件（兜底）

脱敏规则：
- 必须脱：API 密钥、Token、密码、IP 地址
- 保留：项目名、工具名、技术方案、框架版本
