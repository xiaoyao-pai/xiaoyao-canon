# 产品设计方法论（AI 驱动）

> 来源：逍遥派经验沉淀 | 脱敏版

## 核心方法：产品简介先行

不要一上来画原型。先跟 AI 写产品简介——帮自己理清思路。

产品简介迭代路径：
```
v1：模糊想法 → 初版方案
v2：讨论后修正 → 加入新发现的需求
v3：简化架构 → 去掉过度设计
```

每一版 AI 不只是改字，是根据新决策把整个逻辑链重新排布。

## AI 在产品设计中的价值

- **你做决策**，AI 把决策的影响扩散到文档的每个角落
- 人脑做决策强，但追踪一个决策对其他模块的连锁影响，AI 比你快十倍
- AI 不替代你做判断，而是给你做判断所需的全部信息

## 从需求到上线的节奏

```
Day 1: 产品简介 v1-v3（理清方向）
Day 2: 技术方案（架构设计）
Day 3-5: 全栈开发（分 TODO 逐个击破）
Day 6: 部署上线
```

## 开源项目复用方法

1. 先看 Issue 和 Star 趋势，不看 README
2. 让 AI 读一遍项目结构，标注核心模块
3. 用最少依赖先跑起来
4. 跑全量测试，确认哪些能用哪些是空壳
5. 决定保留/替换/删除什么
6. 不要执着于原项目架构——你的场景可能不需要它的前端

## 去 AI 味写作

- 结构改动无效，要逐句定向消除
- 用第一人称叙述、具体时间点、口语化表达
- 避免排比句、对称表格、整数分组
- 三轮 41 处修改后，检测工具无法识别
